# Таблицы

## Диалоги

Переписка между ИИ-агентом и пользователями через Telegram.

### dialogs

- ENV key: `SUPABASE_DIALOGS_TABLE`

Хранит все сообщения чатов. Каждая строка — одно сообщение. Уникальная беседа определяется по tg_chat_id.

#### Поля


| Поле          | Тип           | Описание                                                                                                                                                                      |
| ------------- | ------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| id            | uuid / bigint | Первичный ключ сообщения                                                                                                                                                      |
| tg_chat_id    | text / bigint | Идентификатор чата Telegram — ключ группировки беседы                                                                                                                         |
| tg_username   | text          | Username Telegram-пользователя — отображается как заголовок беседы                                                                                                            |
| sender / role | text          | Отправитель: user — пользователь, agent — ИИ-агент                                                                                                                            |
| content       | text          | Текст сообщения                                                                                                                                                               |
| is_followup   | boolean       | Признак автоматического напоминания после молчания пользователя: true — сообщение добавлено workflow фоллоу-апа (n8n); обычные ответы основного агента учитываются как false. |
| created_at    | timestamptz   | Время создания сообщения                                                                                                                                                      |


#### Примечания

- В SQL сценария фоллоу-апа «последним обычным ответом агента» считается сообщение с role = agent и is_followup = false.

## Фоллоу-ап (молчание пользователя)

Отдельный процесс по расписанию в n8n: напоминание пользователям, которые не ответили после ответа агента. Пороги задаются в escalation_levels; учёт отправленных напоминаний — в unanswered_escalations.

### escalation_levels

Справочник уровней напоминания: для каждой включённой строки задаётся порог задержки; workflow выбирает чаты, где молчание после последнего «обычного» сообщения агента превысило threshold.

#### Поля


| Поле       | Тип      | Описание                                                                                                                                 |
| ---------- | -------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| level      | text     | Идентификатор уровня (используется в журнале unanswered_escalations и в SQL-кандидатах)                                                  |
| threshold  | interval | Интервал PostgreSQL: насколько должно устареть последнее сообщение агента относительно now(), чтобы чат попал в кандидаты на этом уровне |
| is_enabled | boolean  | Учитывать ли уровень при выборе кандидатов                                                                                               |


#### Примечания

- Просмотр через Supabase REST API: GET https://…/rest/v1/escalation_levels?select=* (заголовки apikey и Authorization — ключ проекта).
- Числовые пороги в минутах настраиваются в данных таблицы, не в коде админ-панели.

### unanswered_escalations

Журнал отправленных фоллоу-апов: для связки tg_chat_id + dialogs_id (последнее «обычное» сообщение агента) + level не допускается повторная отправка на том же уровне.

#### Поля


| Поле       | Тип           | Описание                                                                                                   |
| ---------- | ------------- | ---------------------------------------------------------------------------------------------------------- |
| tg_chat_id | bigint / text | Идентификатор чата Telegram                                                                                |
| dialogs_id | uuid / bigint | id строки в dialogs — последнее сообщение агента с is_followup = false, от которого отсчитывается молчание |
| level      | text          | Уровень из escalation_levels                                                                               |


#### Примечания

- Заполняется в n8n узлом INSERT … ON CONFLICT DO NOTHING RETURNING ….

## База знаний (LightRAG)

Ответы на типовые вопросы агент подбирает через инструмент VECTOR к индексу LightRAG. Карточки FAQ редактируются в Notion; workflow n8n переносит их на сервер LightRAG и ведёт журнал в таблице lightrag.

### lightrag

Зеркало записей базы знаний: текст вопроса/ответа для upsert и идентификаторы асинхронной обработки документов в LightRAG (track_id, doc_id). Связана с FAQ в Notion по полю instance_id (стабильный id строки FAQ).

#### Поля


| Поле        | Тип    | Описание                                                                                                              |
| ----------- | ------ | --------------------------------------------------------------------------------------------------------------------- |
| instance_id | text   | Внешний ключ записи FAQ (совпадает с id страницы в потоке n8n; upsert по этому полю)                                  |
| category    | text   | Категория FAQ из Notion                                                                                               |
| content     | text   | Объединённый текст «Вопрос … Ответ …» для учёта в PostgreSQL                                                          |
| track_id    | text   | Идентификатор отслеживания загрузки/сканирования в API LightRAG (пусто — документ ещё не отправлен или пересоздаётся) |
| doc_id      | text   | Идентификатор документа в LightRAG после обработки                                                                    |
| id          | bigint | Суррогатный ключ строки в таблице lightrag                                                                            |


#### Примечания

- Веб-интерфейс инстанса LightRAG: [https://lightrag.nhc.live/webui/#/](https://lightrag.nhc.live/webui/#/)
- Отдельного раздела в админ-панели nhc-admin под эту таблицу нет; данные нужны автоматизации и отладке синхронизации.

## Заявки

Регистрации пользователей на мероприятия.

### event_registrations

- ENV key: `SUPABASE_EVENT_REGISTRATIONS_TABLE`

Каждая строка — отдельная заявка на участие в мероприятии. Ссылка «Диалог» ведёт на соответствующий чат.

#### Поля


| Поле       | Тип           | Описание                              |
| ---------- | ------------- | ------------------------------------- |
| id         | uuid / bigint | Первичный ключ заявки                 |
| tg_chat_id | text / bigint | Telegram chat ID для связи с диалогом |
| created_at | timestamptz   | Дата и время регистрации              |


#### Примечания

- Все остальные колонки подгружаются динамически и отображаются в интерфейсе.
- Способ связи с диалогом задаётся через SUPABASE_EVENT_REGISTRATIONS_DIALOG_LINK_MATCH: chat_id или username.

## Обращения

Сообщения, эскалированные из диалогов для ручной обработки оператором.

### escalation_message

- ENV key: `SUPABASE_ESCALATION_MESSAGE_TABLE`

Эскалация из диалога «Виктории»: строка создаётся инструментом escalation в n8n; затем по расписанию отдельный workflow рассылает необработанные строки операторам в Telegram и помечает status.

#### Поля


| Поле       | Тип           | Описание                                                                                                  |
| ---------- | ------------- | --------------------------------------------------------------------------------------------------------- |
| id         | uuid / bigint | Первичный ключ записи                                                                                     |
| username   | text          | Telegram username клиента (для отображения и ссылки на диалог)                                            |
| message    | text          | Контекст обращения для куратора (полная формулировка задачи)                                              |
| summary    | text          | Сводка для оператора (JSON контекста get-info и/или текст резюме агента)                                  |
| chat_id    | bigint / text | Telegram chat ID беседы — для перехода к диалогу в админке (аналог может называться tg_chat_id в конфиге) |
| status     | boolean       | false — ещё не отправлено в Telegram кураторам; true — обработано циклом по расписанию                    |
| created_at | timestamptz   | Дата и время записи обращения                                                                             |


#### Примечания

- Дополнительные колонки подхватываются в интерфейсе, если есть в таблице.
- SUPABASE_ESCALATION_MESSAGE_DIALOG_COLUMN обычно указывает колонку для ссылки «Диалог» (chat_id или tg_chat_id).

## Заметки о пользователях

Таблица user_profile — журнал дополнительной информации из диалогов: пожелания, личные обстоятельства и другое то, что агент сохранил для сервиса команды.

### user_profile

Не «одна строка профиля на человека», а множество записей на один Telegram username: каждый осмысленный фрагмент (аллергия, пожелание по размещению, ограничения по здоровью и т.д.) может быть отдельной строкой со свободным текстом в description.

#### Поля


| Поле        | Тип           | Описание                                                                                                 |
| ----------- | ------------- | -------------------------------------------------------------------------------------------------------- |
| id          | uuid / bigint | Первичный ключ записи заметки                                                                            |
| username    | text          | Username в Telegram — ключ группировки записей одного пользователя                                       |
| description | text          | Текст заметки: то, что агент передал инструментом user_profile из реплики пользователя или своего резюме |
| created_at  | timestamptz   | Время создания записи (если есть в вашей схеме Supabase)                                                 |


#### Примечания

- Заполняется под-workflow n8n (Supabase «Create a row» из инструмента user_profile).
- Для сводки по человеку читайте несколько строк с одним username или агрегируйте в приложении / SQL-представлении.

## Системные таблицы

Внутренние таблицы Laravel. Хранятся в основной БД приложения.

### users

Пользователи административной панели. Управляются через Laravel Fortify.

#### Поля


| Поле                    | Тип       | Описание                         |
| ----------------------- | --------- | -------------------------------- |
| id                      | bigint    | Первичный ключ                   |
| name                    | varchar   | Отображаемое имя                 |
| email                   | varchar   | Email — используется для входа   |
| password                | varchar   | Хэш пароля (bcrypt)              |
| two_factor_secret       | text      | Секрет для TOTP 2FA (зашифрован) |
| email_verified_at       | timestamp | Время подтверждения email        |
| created_at / updated_at | timestamp | Временны́е метки                 |


### cache / cache_locks

Драйвер кэша Laravel. Хранит временные данные: сессии, throttle-счётчики и другое.

#### Поля


| Поле       | Тип        | Описание                    |
| ---------- | ---------- | --------------------------- |
| key        | varchar    | Уникальный ключ записи кэша |
| value      | mediumtext | Сериализованное значение    |
| expiration | int        | Unix-timestamp истечения    |


### jobs / job_batches / failed_jobs

Очереди задач Laravel. Хранят задачи, ожидающие выполнения, и упавшие задачи с трейсом ошибки.

#### Поля


| Поле                      | Тип      | Описание                               |
| ------------------------- | -------- | -------------------------------------- |
| id                        | bigint   | Первичный ключ                         |
| queue                     | varchar  | Название очереди                       |
| payload                   | longtext | Сериализованный класс задачи с данными |
| attempts                  | tinyint  | Количество попыток выполнения          |
| available_at / created_at | int      | Временны́е метки                       |
